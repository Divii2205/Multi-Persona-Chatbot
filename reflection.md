Reflection on Implementation and Lessons Learned

What worked
The incremental approach of identifying and fixing UI and backend issues proved effective. I first traced the React key warning to index-based keys and updated them to deterministic keys, which immediately removed duplicate-child warnings. Implementing a model fallback chain in `src/services/gemini.ts` increased resilience against API rate limits by sequentially trying alternative models and logging meaningful diagnostics. On the UI side, parsing and rendering of both triple-backtick blocks and inline backtick code in `src/app/page.tsx` made responses far more readable; block code gets a styled container with a copy button, and inline code is displayed with monospace styling. Adding a `conversationStarted` flag fixed the unintended reappearance of suggested questions after clearing the chat, restoring the initial page state behavior users expect.

What GIGO taught me
The GIGO (Garbage In, Garbage Out) principle was a clear reminder that the quality of inputs dictates the usefulness of outputs — especially when integrating external LLM responses. If the model returns poorly formatted text (e.g., inconsistent backtick usage or fragmented code), simply rendering it verbatim results in a poor UX. Instead, treating the model output as untrusted input and applying deterministic parsing and sanitization (detecting code blocks, inline code, and trimming unexpected whitespace) produces a much better result. This also reinforced the need for defensive programming: always validate, normalize, and handle edge cases in external responses before presenting them to users.

What I would improve
Next steps would focus on robustness and polish. First, add syntax highlighting (Prism or Highlight.js) for code blocks to improve readability and context. Second, move parsing and rendering logic into small, testable components and add unit tests for parsing edge cases (nested backticks, missing closing fences, language tags). Third, surface clearer user feedback for fallback behavior — when a fallback model is used, indicate it subtly so advanced users understand why results might vary. Finally, implement streaming responses so the UI can render partial model output progressively and remove the “thinking” indicator exactly when the final content completes. These changes would increase reliability, maintainability, and user confidence in the app.
